I started with 778, that's all I can say to your second question. The
syptoms, well, I compiled like twenty different versions, and set the
iBook sleep to 1 minute. The results vary from my first statement:

Not allowing for sleep, but working perfectly otherwise:

  ./configure --prefix=/usr/local --enable-nntp --enable-compressed
  --enable-pop --enable-imap --with-ssl --with-gss
  --with-docdir=/usr/local/share/doc/mutt --sysconfdir=/usr/local/etc
  --enable-hcache --with-libesmtp=/usr/local 

Allowing for sleep:

  ./configure --prefix=/usr/local --enable-nntp --enable-compressed
  --enable-pop --enable-imap --with-ssl --with-gss
  --with-docdir=/usr/local/share/doc/mutt --sysconfdir=/usr/local/etc
  --enable-hcache --with-libesmtp=/usr/local --enable-buffy-size

The latter, used with "set force_buffy_check=yes", seems to work most
of the time. Particularly, there are problems when a mailbox receives
new mail, I switch to it and read all mail, then read news, and a
different mailbox receives mail. In this case, the first mailbox is
marked as containing new mail even though this isn't true -- which is
the typical problem --enable-buffy-size used to cure on upstream
mutt-dev. With "set force_buffy_check=no", it always show the
erroneous buffy behavior.

I'm evaluating a macro <sync-mailbox><change-newgroup> to see if that
makes a difference or not. But of course it would be working around
the problem.

This is an iBook G3 900 w/ Mac OS X 10.4.4 with HFS+/journaled and
FileVault here.
