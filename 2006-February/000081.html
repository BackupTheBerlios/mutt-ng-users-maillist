> First of all, what status does the mail stored in your INBOX have? Is it new or marked read already?

It's always marked as NEW.

> Second, what are your values for the following variables: $record, $save_name and $force_name? The folder you name 
> via $record (or the others mentioned plus fcc-hook) is shown as the 'Fcc:' pseudo header in the compose screen. What 
> does it say?

Verbatim from my muttngrc file:

  set record='+.SENT'
  unset save_name            
  unset force_name           

and no entry with anything matching fcc.

The result of muttng -t yields the absolute path of my SENT directory:

record="/home/ydef/Maildir/.SENT"

and the precise values I had set in my config file, which also happen to be the muttng defaults for these values: 

save_name=no
fcc_attach=yes
force_name=no
