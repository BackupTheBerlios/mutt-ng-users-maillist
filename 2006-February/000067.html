If so, I completely misunderstood the problem. When muttng (as well as 
mutt btw) store sent mail via $record/fcc-hook/etc., they store the mail 
directly to the destination folder. Read: it never gets inserted into 
any filtering chains such as procmail so tweaking procmail shouldn't 
have any effect if the problem was mutt(ng)-only. This procmail solution 
only works when someone's own mails come back via a mailing list so that 
procmail marks them as read (it could mark any other mail as read, not 
just the own).

The exception would be that someone uses cronjobs to pick up mails from 
local folders and inserting them into procmail to filter again (but I 
cannot imagine someone really does something similar).

So are we talking about "coming-back" mails not being marked as read or 
mails really stored via $record/fcc-hook not marked as read?
