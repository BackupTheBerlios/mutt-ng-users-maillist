My ~/.procmailrc sorts mail based on the Delivered-To: and List-Id:
headers, creating new ~/Mail/lists/... maildirs on demand. The following
settings work on mutt-ng’s start:

mailboxes =unsure `echo /home/shot/Mail/lists/*`
lists `cd /home/shot/Mail/lists/; echo *`

I use mutt-ng in a screen on a remote server, though,
so I don’t restart it for days. What I’d like to have is

folder-hook . mailboxes =unsure `echo /home/shot/Mail/lists/*`

This is evalated once on mutt-ng start, but doesn’t seem to be
re-evaluated on folder changes. I’m using Debian’s 0.0+20060113-1
package (r774), rebuilt for Ubuntu Breezy.
