I get the "bad record mac" error with muttng using OpenSSL _and_ GnuTLS. 
Also, in the past we had a bug report that gmane doesn't support an NNTP 
extension other providers support for quite some years now.

So I guess this is a gmane issue. I don't have another SSL-enabled 
newsreader at hand to test this.

Using 'gnutls-cli-debug -p 563 snews.gmane.org' it prints that the 
server doesn't support SSL3 and TLS 1.0 as well as 1.1. This means that 
it doesn't work with GnuTLS (what you're trying).

With OpenSSL, there're many ssl_* variables. When using SSLv2 (v3 is not 
supported) I get another error 'bad mac decode' and 'bad mac record' 
with v3.

My conclusion: this is a gmane problem.

connect to gmane via NNTPS with another newsreader?
